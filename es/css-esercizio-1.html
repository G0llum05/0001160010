üìñ Teoria

Position: Static

Il posizionamento predefinito per tutti gli elementi. Gli elementi appaiono nell'ordine in cui sono scritti nel codice HTML.

div {
  position: static; /* default */
}

Position: Relative

L'elemento viene posizionato secondo il normale flusso del documento e poi spostato rispetto a se stesso usando top, right, bottom, left. Lo spostamento non influisce sulla posizione di altri elementi.

div {
  position: relative;
  top: 20px;
  left: 10px;
}

Position: Absolute

L'elemento viene rimosso dal flusso normale e posizionato rispetto al suo primo antenato posizionato (non static). Se non esiste, si posiziona rispetto al documento.

div {
  position: absolute;
  top: 50px;
  right: 20px;
}

Position: Fixed

Simile ad absolute, ma l'elemento rimane fisso rispetto alla viewport anche quando si scrolla la pagina.

div {
  position: fixed;
  bottom: 0;
  right: 0;
}

Position: Sticky

Un ibrido tra relative e fixed. L'elemento si comporta come relative finch√© non raggiunge una soglia di scroll, poi diventa fixed.

div {
  position: sticky;
  top: 0;
}

üí° Nota Importante

Per usare position: absolute correttamente, l'elemento genitore deve avere position: relative (o absolute/fixed). Questo crea un "contesto di posizionamento".
